# План: Настройка логирования PostgreSQL + pg_stat_statements + auto_explain

## Обзор

Добавляем мониторинг производительности PostgreSQL на трёх уровнях:
1. **Логирование медленных запросов** (формат, совместимый с PGBadger)
2. **pg_stat_statements** — агрегированная статистика всех запросов
3. **auto_explain** — автоматические EXPLAIN-планы для медленных запросов

**Go-код практически не меняется.** Все настройки — на стороне PostgreSQL.

---

## Порог медленного запроса: 200 мс

Обоснование:
- User-service: простые CRUD (< 5 мс)
- Appeal-service: простые CRUD (< 5 мс)
- Board-service: JOIN 4+ таблиц, JSON-агрегация (10–100 мс в норме)
- 200 мс = двукратный запас над самой тяжёлой нормой
- Пользователь фронтенда чувствует задержку > 200 мс

Для `auto_explain` порог выше: 500 мс (чтобы не дублировать обычные логи).

## Содержимое `deployments/nexus/configs/postgresql.conf`

```ini
# Logging Destination
log_destination = 'stderr'
logging_collector = on
log_directory = 'pg_log'
log_filename = 'postgresql-%Y-%m-%d_%H%M%S.log'
log_rotation_age = 1440
log_rotation_size = 102400
log_truncate_on_rotation = on

# What to Log
log_min_duration_statement = 200
log_checkpoints = on
log_connections = on
log_disconnections = on
log_lock_waits = on
log_temp_files = 0
log_error_verbosity = default

# PGBadger-compatible log line prefix
log_line_prefix = '%m [%p]: [%l-1] user=%u,db=%d,app=%a,client=%h '

# pg_stat_statements
shared_preload_libraries = 'pg_stat_statements,auto_explain'
pg_stat_statements.track = all
pg_stat_statements.track_utility = on
pg_stat_statements.max = 10000

# auto_explain
auto_explain.log_min_duration = 500
auto_explain.log_analyze = on
auto_explain.log_buffers = on
auto_explain.log_timing = on
auto_explain.log_triggers = on
auto_explain.log_verbose = on
auto_explain.log_nested_statements = on
auto_explain.log_level = 'LOG'
```

## Изменения в Kubernetes `postgres.yaml`

```yaml
containers:
  - name: {{ $svcName }}
    image: {{ $.Values.postgres.image }}
    args: ["-c", "config_file=/etc/postgresql/postgresql.conf"]
    volumeMounts:
      - name: postgres-data
        mountPath: /var/lib/postgresql/data
      - name: postgres-config
        mountPath: /etc/postgresql/postgresql.conf
        subPath: postgresql.conf
        readOnly: true
    env: ...
volumes:
  - name: postgres-data
    persistentVolumeClaim:
      claimName: ...
  - name: postgres-config
    configMap:
      name: postgres-config
```

## Изменения в Go-коде (application_name)

Только в одном месте на сервис — строка DSN в `setupPostgresPool()`:

```go
// Было:
DSN := fmt.Sprintf("postgres://%s:%s@%s:%s/%s?sslmode=disable", ...)

// Стало:
DSN := fmt.Sprintf("postgres://%s:%s@%s:%s/%s?sslmode=disable&application_name=nexus-board", ...)
```

Аналогично для appeal (`nexus-appeal`) и user (`nexus-user`, в `app.go`).
