# Plan: Перенос VK OAuth бизнес-логики в User микросервис

## Текущее состояние

VK OAuth flow разделен между двумя микросервисами:
- **Authorization** — обменивает `code` на `access_token + email` (`ExchangeVKCode`)
- **User** — по `access_token` делает VK API запрос для получения `first_name`, затем ищет/создает юзера (`ProcessUserWithVK`)

Фасад вызывает оба сервиса последовательно: `ExchangeVKCode` → `ProcessUserWithVK` → `CreateSession`.

## Целевое состояние

Вся бизнес-логика VK OAuth — в User микросервисе:
- **Authorization** — только управление сессиями (убрать `ExchangeVKCode`)
- **User** — принимает `code`, `code_verifier`, `state`, `device_id`, делает полный цикл (обмен кода → получение данных пользователя → find-or-create user) и возвращает `user_link`
- **Фасад** — только валидация входных параметров и проброс в User сервис, затем создание сессии через Auth сервис
